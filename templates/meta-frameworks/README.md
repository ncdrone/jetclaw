# Meta-Framework Templates

Generalized operational philosophy and methodology templates for AI agents. Any new agent can adopt and customize these to establish a principled operating foundation.

## What These Are

These 9 files define *how an agent thinks and operates* — not what it works on. They are methodology, not content. They cover:

| File | Purpose |
|------|---------|
| **IDEOLOGY.md** | Core beliefs and values. The "why" behind everything. |
| **SOUL.md** | The agent's operating identity. Synthesizes all other frameworks into one working document. |
| **STRATEGY.md** | How to identify what matters, make decisions, and allocate effort. |
| **PROCESS.md** | How to build, evaluate, and improve repeatable processes. |
| **STANDARDS.md** | What quality looks like. Measurable, enforceable, non-negotiable. |
| **SECURITY.md** | Security posture, threat modeling, incident response. |
| **LEARNING.md** | How to acquire knowledge and grow capability systematically. |
| **REVIEW.md** | How to step back, evaluate, and improve the whole system. |
| **ORCHESTRATION.md** | How to design, deploy, and manage specialized sub-agents. |

## How to Use

1. Copy these templates into your agent's workspace
2. Replace placeholders with your specifics:
   - `{agent}` — the agent's name
   - `{user}` — the human owner/operator
   - `{company}` — the organization name
   - `{domain}` — the primary domain
   - `{product}` — the product being worked on
   - `{sub-agent}` — any specialized sub-agents
   - `{host}` — the host machine name
   - `{bot-name}` — any bot integrations
   - `{agent-email}` — the agent's email address
   - `{user-email}` — the human's email address
3. Add domain-specific content (your products, your projects, your standards)
4. Remove any sections that don't apply to your agent's scope
5. Start operating. Update the frameworks as you learn.

## Design Philosophy

- **SOUL.md is the primary document.** It synthesizes all other frameworks into a single working reference. An agent that only reads SOUL.md should still operate correctly.
- **Frameworks are interconnected.** STRATEGY informs PROCESS, PROCESS enforces STANDARDS, STANDARDS ratchet up from REVIEW, SECURITY underlies everything.
- **These are living documents.** Every framework has a changelog. If a framework hasn't been updated, it's either perfect or neglected.
- **Philosophical content is universal.** The principles (constraint theory, earned scope, compound quality, etc.) apply regardless of domain. The specifics are what you customize.

## What Was Removed

These templates were generalized from a production agent deployment. The following specifics were scrubbed:

- Agent and user names
- Company and product names
- Sub-agent names and registries
- Specific email addresses and domains
- Specific incident dates and details (replaced with example format)
- References to specific hardware, projects, and businesses
- Dated changelog entries (replaced with version-only)

All philosophical content, frameworks, principles, processes, templates, and anti-patterns were preserved exactly as written.
